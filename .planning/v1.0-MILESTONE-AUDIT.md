### Integration Finding 1: walkVariable Predicate Gap (Non-Critical)

**Severity:** Non-critical (narrow edge case)
**Affected Requirements:** EXPR-03, ADV-02

When a `VariableNode` with predicates appears as a **standalone function argument** (not as a step inside a PathNode), `walkVariable` resolves the variable but never inspects `node.predicate`. Phase 6 fixed the symmetric case in `walkPath`'s variable-resolution branch but did not update `walkVariable`.

| Case | Behavior |
|------|----------|
| `orders[$field].price` (direct NameNode) | ADV-02 fires → `orders[*]` emitted |
| `($data := orders; $data[$field].price)` (VariableNode in PathNode) | Phase 6 fix fires → `orders[*]` emitted |
| `$map($data[status], fn)` (standalone VariableNode) | walkVariable skips predicates → filter paths dropped |

### Integration Finding 2: CLI Error Message (Non-Critical)

**Severity:** Non-critical (UX degradation)
**Affected Requirement:** API-02

`src/cli.ts` uses `err instanceof Error` but jsonata throws plain objects (not `Error` instances). The fallback `String(err)` produces `"Error: [object Object]"` instead of the actual error message. Fix: access `err.message` directly.
