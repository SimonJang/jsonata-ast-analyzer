## Tech Debt

### Phase 2: Scope Infrastructure and Variable Tracing

- **OBS-1:** `$sort` has higher-order semantics defined in `HIGHER_ORDER_SEMANTICS` but no test coverage. If `$sort` lambda handling regresses, no test will catch it. Affects SCOPE-04.
- **OBS-2:** `walkPath` drops name-prefix steps before a variable step (e.g., `accounts.$x.field` discards `accounts`). Documented known limitation from Phase 2 research.
- **OBS-3:** Standalone `BindNode` outside a block reaches `walkBind` but only returns RHS paths without calling `bindVariable`. Code path is reachable but untested.
- **OBS-4:** `$lookup` is in `BUILTIN_FUNCTIONS` but not in `HIGHER_ORDER_SEMANTICS` — falls through to pass-through argument handler (over-approximates paths).
