2. **Complex multi-variable expression:** `($x := account.name; $y := account.email; {"name": $x, "email": $y})` -- multiple static paths through variable resolution. Use shell escaping: pass expression wrapped in double quotes with inner `$` signs escaped, OR use the approach of writing the expression to a temp string. Recommended approach: use `execSync` with the expression as a single-quoted shell argument. Since the expression contains `$` which the shell will try to expand, pass it carefully. Best approach: `execSync(\`node dist/cli.js '\${expr}'\`, ...)` will NOT work because single quotes in template literals still get shell-expanded. Instead, write the expression directly in the execSync call with proper escaping, or use `JSON.stringify` and pass via echo pipe: `echo ${JSON.stringify(expr)} | node dist/cli.js` -- but CLI reads from argv not stdin. Safest approach for expressions with `$`: use `child_process.execFileSync('node', ['dist/cli.js', expr])` which bypasses shell entirely. Import `execFileSync` instead of `execSync` from `node:child_process`.

3. **Dynamic confidence expression:** `items[condition].name` -- produces both static and dynamic paths. Compare CLI JSON output with API output.

it("CLI round-trip: [description]", () => {

  const apiResult = extractPaths(expr);
  expect(sortPaths(cliResult)).toEqual(sortPaths(apiResult));
});
