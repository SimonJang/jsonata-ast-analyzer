    1. In `src/walker.ts`, in the `walkUnary` function, replace the `"["` case (currently line ~364-366):

    **Current code:**
    ```typescript
    case "[":
      return (node.expressions ?? []).flatMap((e) => walkNode(e, scope));
    ```

    **Replace with sequential scope accumulation (same pattern as `walkBlock`):**
    ```typescript
    case "[": {
      // Array constructor with sequential scope accumulation
      // (mirrors walkBlock pattern: bind expressions update scope for subsequent elements)
      const paths: string[] = [];
      let currentScope = scope;
      for (const expr of node.expressions ?? []) {
        if (expr.type === "bind") {
          const bindNode = expr as BindNode;
          const rhsPaths = walkNode(bindNode.rhs, currentScope);
          paths.push(...rhsPaths);
          currentScope = bindVariable(currentScope, bindNode.lhs.value, rhsPaths);
        } else {
          paths.push(...walkNode(expr, currentScope));
        }
      }
      return paths;
    }
    ```

    **Key points:**
    - Uses block scope (`{ }`) for the case to allow `const` declarations
    - `currentScope` starts as the passed-in `scope` (no child scope needed -- same as `walkBlock` pattern)
    - When a `bind` expression is encountered, its RHS paths are walked and the variable is bound in `currentScope` for subsequent expressions
    - Non-bind expressions are walked with the accumulated `currentScope`, allowing variable references to resolve
    - The accumulated scope stays local to this function call (returned value is `paths`, not scope)

    **Pitfall 5 prevention:** The accumulated scope does NOT leak upward because `walkUnary` returns `string[]` (paths), not a scope. The `currentScope` variable is local to this case block. Parent scope is unmodified.
