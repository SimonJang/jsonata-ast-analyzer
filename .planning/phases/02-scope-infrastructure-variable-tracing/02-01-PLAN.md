### SCOPE-01: Variable assignment tracing
- `extractPaths('($x := account.name; $x)')` -> `[{ path: "account.name" }]`
  Variable $x resolves to the assigned path; the overall output includes "account.name" once (deduped from bind RHS + variable use).
- `extractPaths('($x := a; $x := b; $x)')` -> `[{ path: "a" }, { path: "b" }]`
  Last-write-wins for resolution ($x resolves to "b"), but "a" was still read from input.
- `extractPaths('($a := x.y; $b := $a.z; $b)')` -> `[{ path: "x.y.z" }]`
  Multi-hop chain: $a resolves to ["x.y"], $b resolves to ["x.y.z"], output is deduped "x.y.z" (because "x.y" is a prefix of "x.y.z" -- but actually both should appear since x.y was read for $a's RHS). Correction: output is `[{ path: "x.y" }, { path: "x.y.z" }]` because $a's RHS reads "x.y", and $b's resolution of $a.z produces "x.y.z". Both are distinct paths.
  Actually, re-analyzing: walkNode on `$a`'s rhs (path x.y) returns ["x.y"]. That goes into the collected paths. Then walkNode on `$b`'s rhs ($a.z) resolves $a to ["x.y"] and appends ".z" -> ["x.y.z"]. That goes into collected paths. Then $b reference resolves to ["x.y.z"]. Deduped: ["x.y", "x.y.z"].
  -> `[{ path: "x.y" }, { path: "x.y.z" }]`
- `extractPaths('($x := a; ($x := b; $x); $x)')` -> `[{ path: "a" }, { path: "b" }]`
  Inner block shadows $x=b but outer $x still resolves to "a". Both "a" and "b" are read from input. Inner $x resolves to "b", outer $x resolves to "a".
- `extractPaths('($x := account; $x.name)')` -> `[{ path: "account" }, { path: "account.name" }]`
  Variable in path context: $x resolves to ["account"], then .name is appended -> "account.name". Plus "account" was read for $x's RHS.

### SCOPE-01: Cycle detection
- `extractPaths('($x := $x.y; $x)')` -> should include `[{ path: "x.y" }]` or similar without infinite recursion
  The $x on the RHS is unresolvable (not yet bound when evaluated). So $x.y produces... $x is unresolvable, so the RHS path extraction for `$x.y` depends on whether $x resolves. Since $x is being defined, it's not yet in scope, so $x is unresolvable -> silent skip -> $x.y produces no paths from the variable part. BUT the AST for `$x.y` is a path with steps [$x, y]. The variable step $x is unresolvable, so we drop the entire path (per discretion decision: drop). RHS returns []. So $x binds to []. Then $x reference returns []. Overall: [].
  -> `[]`
- `extractPaths('($a := $b; $b := $a; $a)')` -> `[]`
  Mutual reference: $a binds to $b's paths (unresolvable at that point -> []), $b binds to $a's paths ([] from above -> []), $a resolves to [].

### SCOPE-02: Context variable binding (@$v)
- `extractPaths('items@$v')` -> `[{ path: "items" }]`
  The @$v binds variable $v to the element context of "items". The path "items" is extracted. The binding itself produces no additional paths -- it just makes $v available in scope for downstream use. (Full resolution of $v.field expressions is tested when combined with filters in Phase 3.)

### SCOPE-03: Positional variables (#$i)
- `extractPaths('items#$i')` -> `[{ path: "items" }]`
  The #$i binds variable $i to the index position. It is a non-data-path reference. Only "items" is extracted as a data path.

### EXPR-05: Function argument extraction
- `extractPaths('$sum(items.price)')` -> `[{ path: "items.price" }]`
  Built-in function call: extract paths from arguments, ignore the function name itself.
- `extractPaths('$count(orders)')` -> `[{ path: "orders" }]`
- `extractPaths('$substring(name, 0, 5)')` -> `[{ path: "name" }]`
  Non-path arguments (numbers) produce no paths.
- `extractPaths('$unknownFunc(a.b)')` -> `[{ path: "a.b" }]`
  Unknown functions: extract arguments as pass-through data paths (over-approximation).
- `extractPaths('$sum(items.price) + $count(orders)')` -> `[{ path: "items.price" }, { path: "orders" }]`
  Multiple function calls in binary expression.

### Edge cases
- `extractPaths('$')` -> `[]`
  Dollar sign alone is a reference to the root input -- currently treated as a variable. Should produce no paths (it's the whole input, not a specific path).
- `extractPaths('($x := 42; $x)')` -> `[]`
  Variable bound to literal: $x resolves to [] (no paths from literal RHS), $x reference returns [].
- `extractPaths('($x := a > 0 ? b.c : d.e; $x)')` -> `[{ path: "a" }, { path: "b.c" }, { path: "d.e" }]`
  Conditional RHS: $x binds to ["b.c", "d.e"] (over-approximate both branches). "a" read in condition. $x resolves to ["b.c", "d.e"]. Deduped overall: ["a", "b.c", "d.e"].

### Step 2: Create src/scope.ts -- Immutable scope chain

```typescript
/**
 * Immutable scope chain for variable resolution.
 * Each scope level has its own bindings and a parent pointer.
 * Lookups walk up the chain until found or exhausted.
 */
export interface ScopeTracker {
  readonly bindings: ReadonlyMap<string, readonly string[]>;
  readonly parent: ScopeTracker | null;
}

const EMPTY_SCOPE: ScopeTracker = { bindings: new Map(), parent: null };

/** Create a new root scope (no parent). */
export function createScope(): ScopeTracker {
  return EMPTY_SCOPE;
}

/** Create a child scope inheriting from parent. */
export function childScope(parent: ScopeTracker): ScopeTracker {
  return { bindings: new Map(), parent };
}

/**
 * Bind a variable name to resolved paths in the current scope level.
 * Returns a NEW scope (immutable -- does not modify input).
 * Last-write-wins: replaces any existing binding at this scope level.
 */
export function bindVariable(
  scope: ScopeTracker,
  name: string,
  paths: readonly string[],
): ScopeTracker {
  const newBindings = new Map(scope.bindings);
  newBindings.set(name, paths);
  return { bindings: newBindings, parent: scope.parent };
}

/**
 * Resolve a variable name by walking up the scope chain.
 * Returns the bound paths or null if unresolvable.
 *
 * Uses a `resolving` set for cycle detection: if the variable
 * is already being resolved (circular reference), returns null.
 */
export function resolveVariable(
  scope: ScopeTracker,
  name: string,
): readonly string[] | null {
  let current: ScopeTracker | null = scope;
  while (current !== null) {
    if (current.bindings.has(name)) {
      return current.bindings.get(name)!;
    }
    current = current.parent;
  }
  return null; // unresolvable
}
```

For the block handler, use a for-loop instead of flatMap:
```typescript
function walkBlock(node: BlockNode, scope: ScopeTracker): string[] {
  const paths: string[] = [];
  let currentScope = scope;
  for (const expr of node.expressions) {
    if (expr.type === "bind") {
      const bindNode = expr as BindNode;
      const rhsPaths = walkNode(bindNode.rhs, currentScope);
      paths.push(...rhsPaths);
      currentScope = bindVariable(currentScope, bindNode.lhs.value, rhsPaths);
    } else {
      paths.push(...walkNode(expr, currentScope));
    }
  }
  return paths;
}
```
